CommonJS 模块管理机制

AMD : require
CMD : sea.js
CommomJS : node.js
ES6 Module
这些模块化思想,规定了我们在JS中我们的模块该如何的创建,如何的导入以及如何导出

  • 内置模块: NODE自带的
    + http/https  创建和管理服务的模块
+ 方式给予JS进行I/O操作的
+ url  解析URL地址的
+ path 管理路径的
+ ...
  • 第三方模块: 基于NPM安装,别人写好给我们用的
    + less
+ mime
+ qs  
+ express
+ express-session
+ body-parser
+ .....
  • 自定义模块 : 自己写的模块

NODE中的模块管理

  1. 在NODE环境下,我们没创建一个JS 都相当于创建了一个新的模块: 模块中的方法也都是模块的私有犯法,不同模块之间的同名方法不会有任何冲突;
  2. module.exports 就是NODE天生自带的用来导出模块中方法的方式

3 require 是NODE天生提供的用来导入模块的方法

  1. 导入模块是同步的(没有导入完成,后面的事情是不处理的); 每一次导入模块,都是吧导入的模块的JS代码从上到下执行一遍
